Final project: two-part build

Your final project has two parts: a group website redesign that continues your midterm work, and an individual personal website you can use beyond this class. Both must be live and published.

Group Project

Website Redesign

In groups of up to 4, redesign the same website your group evaluated for the midterm. Your goal is to deliver a clearly improved, professional version.

Core requirements

  • Use the same approved website from your midterm
  • No commercial templates. Bootstrap is allowed.
  • Use valid HTML and CSS with readable, organized code
  • Design must be responsive with no horizontal scrolling
  • Use clean file names and organized folders
  • Publish live using GitHub Pages or CodePen multi-page format

Show clear improvement

  • Stronger layout and visual design
  • Better navigation and readability
  • Better spacing, hierarchy, and imagery
  • Improved mobile experience

Submit as a group

  • Bullet-point summary of improvements
  • Live site link
  • About 5-minute video: original problems, redesign choices, and improvements
Individual Project

Personal Website

Build a personal website you can use for internships, jobs, or showcasing your work and interests.

Core requirements

  • Build with HTML and CSS. Bootstrap is allowed.
  • No commercial templates
  • Include at least 5 pages with different layouts
  • Header and footer may stay consistent
  • Site must be responsive with no horizontal scrolling
  • Use organized folders and clean file names
  • Publish live using GitHub Pages or CodePen multi-page format

Content expectations

  • Combine text, images, links, and structure
  • Avoid empty pages or pages made only of bullets
  • Make it something you would genuinely show an employer

What matters most in modern web development?

Modern web development is more than getting code to work. It means building websites that are readable, responsive, interactive, accessible, organized, and ready to publish.

Foundation

Semantic HTML

Use meaningful elements like header, nav, main, section, and footer so your content has structure and purpose.

Design

Responsive Layouts

Your site should work on phones, tablets, and desktops instead of only looking right on your own screen.

Layout

Flexbox and Grid

Use modern CSS layout tools to create clean, adaptable page structures.

Interaction

JavaScript

Use JavaScript to make your site respond to clicks, inputs, forms, menus, and dynamic changes.

Inclusion

Accessibility

Think about readable text, alt text, keyboard use, contrast, and logical structure.

Publishing

Version Control + Deployment

A strong project is organized, backed up, and shareable through a live link.

CodePen vs. GitHub Pages

Both can work. The best choice depends on whether you want the fastest route or the more portfolio-ready workflow.

Low friction

CodePen

Best for: Fast prototyping, quick demos, smaller front-end builds, and students who want less setup.

  • Very easy to start
  • Live preview updates quickly
  • Great for experimenting with HTML, CSS, and JavaScript
  • Helpful for one-page builds or front-end demos
Portfolio-ready

GitHub Pages

Best for: Multi-page sites, portfolios, stronger file organization, and more realistic publishing practice.

  • Great for static final websites
  • Builds familiarity with repositories and file structure
  • Easy to share as a live project URL
  • Closer to how many developers publish front-end work

What an A-level project looks like

Use this section to compare your work against a stronger final project standard. This is not about being flashy. It is about being intentional, polished, and usable.

Needs work

Lower-quality final project

  • Looks unfinished or rushed
  • Weak spacing, alignment, or inconsistent styling
  • Navigation is confusing or incomplete
  • Pages feel empty, repetitive, or thin on content
  • Little or no evidence of thoughtful mobile design
  • Broken links, missing images, or messy file structure
  • JavaScript is missing, minimal, or feels disconnected
  • Does not feel like something the student would want to show publicly
A-level target

Higher-quality final project

  • Clear purpose and audience from the start
  • Strong visual hierarchy and consistent design choices
  • Navigation is simple, complete, and easy to use
  • Content feels thoughtful, complete, and well structured
  • Responsive layout works cleanly on smaller screens
  • Images, text, colors, and spacing feel intentional
  • JavaScript adds meaningful interactivity or usefulness
  • Feels polished enough to include in a portfolio or interview conversation
Weak layout example

Messy / low-quality design

  • No spacing or alignment
  • Everything feels cramped
  • No visual hierarchy
  • Hard to scan
Strong layout example

Clean / A-level design

  • Clear spacing and alignment
  • Strong visual hierarchy
  • Easy to scan content
  • Feels intentional and polished
Personal site

A-level signs

A strong personal website feels complete, visually consistent, and clearly about the student. It has enough content to feel real, not filler.

Redesign project

A-level signs

A strong redesign makes the original site noticeably better. The improvement is visible in layout, readability, usability, and mobile experience.

Step-by-step publish paths

Pick the path that matches your recommendation or your comfort level.

Best for multi-page and portfolio-ready sites

GitHub Pages walkthrough

  1. Create a GitHub account
  2. Create a repository
  3. Add index.html, CSS, JS, and images
  4. Commit your changes
  5. Open Settings and turn on GitHub Pages
  6. Choose the publishing source
  7. Test the live link on desktop and mobile
Common mistakesMissing index.html, broken paths, wrong capitalization, and forgetting to republish after changes.
Best for fast and simple front-end builds

CodePen walkthrough

  1. Create a CodePen account
  2. Start a new Pen
  3. Build and preview as you go
  4. Keep your code clean
  5. Test responsiveness and readability
  6. Save and share the link
Best use caseA one-page site, front-end demo, creative interaction, or smaller project where speed matters most.
Worth knowing

Other modern publishing options

You may also hear about tools like Netlify and Vercel. They are useful platforms, but for this class, GitHub Pages and CodePen are the most straightforward options.
