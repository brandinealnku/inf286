Final project: two-part build

Your submission still includes a group redesign and an individual site. The difference now: you will execute both with current UX best practices so the experience feels intentional, not just complete.

Group project

Website Redesign

Redesign the same approved website from your midterm analysis and demonstrate measurable usability improvements.

Execution priorities

  • Keep navigation predictable and label links clearly
  • Use responsive layout patterns with no horizontal scroll
  • Improve readability (contrast, spacing, chunked content)
  • Use semantic HTML and valid CSS
  • Publish live on GitHub Pages or CodePen multi-page setup
Individual project

Personal Website

Build a personal site with clear positioning, project proof, and calls to action for internships/jobs.

Execution priorities

  • At least 5 pages with different layout intent
  • Consistent visual hierarchy on every page
  • Meaningful content blocks (not filler text)
  • Accessible interactions and focus states
  • Live published URL + clean organization

What matters most in current web UX?

Today’s successful pages tend to follow the same patterns: clear structure, reduced cognitive load, visible trust signals, and fast action paths on mobile and desktop.

Clarity

Strong information hierarchy

Headings, spacing, and progressive disclosure help users decide what to read next in seconds.

Confidence

Credibility + transparency

Users convert more when pages show clear purpose, ownership, and trustworthy detail.

Inclusion

Accessible interactions by default

Visible focus, keyboard support, and adequate contrast improve outcomes for everyone.

CodePen vs. GitHub Pages

Choose the platform based on outcome: speed for short demos, structure for portfolio-ready builds.

CodePen

Long-term value

GitHub Pages

  • Better multi-page organization
  • Closer to real-world workflow
  • Stronger showcase for portfolios

Step-by-step publish paths

Pick the path that best matches your recommendation and timeline.

Best for multi-page portfolio-ready sites

GitHub Pages walkthrough

  1. Create GitHub account and repository
  2. Add organized project files
  3. Commit with clear messages
  4. Enable GitHub Pages in Settings
  5. Select source branch and folder
  6. Verify all links/images on mobile + desktop
Common mistakes

Missing index.html, broken relative paths, capitalization mismatches, and not waiting for publish refresh.

Best for rapid prototypes and demos

CodePen walkthrough

  1. Create account and new Pen
  2. Build in HTML/CSS/JS panels
  3. Keep structure semantic and readable
  4. Test at mobile widths
  5. Save and publish shareable link
Best use case

One-page experiences, interactions, or light front-end projects where setup speed matters.

Worth knowing

Other modern hosting options

Netlify and Vercel are common in industry, but GitHub Pages + CodePen remain the most straightforward options for this course workflow.
