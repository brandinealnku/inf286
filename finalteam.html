Assignment overview in plain language

What your team is trying to accomplish.

The goal

Redesign the same website your group selected for the midterm. Your new version should make the site look better, feel easier to use, read more clearly, and work better on mobile.

What counts as meaningful improvement
  • Stronger visual design and layout
  • Better navigation and usability
  • Clearer, easier-to-read content
  • Better spacing, structure, and image choices
  • A noticeably improved mobile experience
Project requirements
  • No commercial or pre-made templates as the final design
  • Bootstrap is allowed
  • Use valid HTML and CSS
  • No broken or inconsistent links
  • Use a clean file/folder structure
  • No horizontal scrolling

Team success checklist

Check these off as your team gets closer to done.

  • Confirmed the original website
    Your redesign matches the same site from the midterm.
  • Assigned team roles
    Examples: project lead, front-end dev, designer, content editor.
  • Tested desktop, tablet, and mobile
    Everything should look solid at more than one screen size.
  • Published the site
    GitHub Pages or CodePen multi-page project link works in incognito.
  • Recorded the final video
    Show the original problems, the redesign, and why it is better.

Suggested project timeline

A realistic plan so your group is not doing everything at the last second.

  1. Step 1: Review the original site. Use your midterm analysis to identify the biggest design and usability problems.
  2. Step 2: Split the work. Decide who handles each page, section, or responsibility.
  3. Step 3: Build the redesign. Focus on layout, readability, consistency, and responsiveness.
  4. Step 4: Test and polish. Click everything. Resize everything. Fix spacing and broken links.
  5. Step 5: Publish and present. Launch the site, record the video, and double-check the final submission.

How to turn this into true live collaboration with Firebase

This version works immediately on GitHub Pages. Right now it saves to the browser with localStorage. Use the steps below if you want the whole group to see shared tasks and notes live.

What is already working right now

This page already supports:

  • Light and dark mode toggle
  • Countdown to May 5 at 11:59 PM
  • Task board with drag and drop
  • Saved notes, checklist progress, and resource links
  • GitHub Pages-friendly static hosting
What to do next for shared live collaboration
  1. Create a Firebase project.
  2. Enable Firestore Database.
  3. Replace localStorage logic with Firestore reads/writes for tasks, notes, checklist items, and links.
  4. Add your Firebase config where indicated in your future version.
  5. Deploy the updated file to GitHub Pages.

A good next step is to store team data under a collection like teams/{teamId}/tasks.

Suggested team roles
  • Project Lead: keeps the team on track and owns the final check
  • Designer: handles layout, typography, colors, spacing, and visual consistency
  • Front-End Developer: codes the structure and styling
  • Content/QA Lead: improves clarity, checks links, tests mobile, and helps with the video

What usually helps teams earn a stronger grade

Keep these in mind while you build.

Make real improvements. Do not just change colors. Improve structure, readability, usability, and mobile experience.
Design on purpose. Strong spacing, contrast, visual hierarchy, and consistency make a site feel more professional.
Keep your code clean. Good file names, indentation, and organization matter.
Test before you submit. Open the live link in incognito, resize the browser, and click everything.
